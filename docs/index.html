Project Overview

This project demonstrates automated login testing for the OrangeHRM demo application using Selenium WebDriver with TestNG framework, integrated with GitHub Actions for continuous integration.

Getting Started

Prerequisites

  • Java JDK 11 or higher
  • Maven 3.6 or higher
  • Google Chrome browser
  • Git

Local Setup

1

Clone the Repository

git clone https://github.com/your-username/demo-pipeline.git
cd demo-pipeline
2

Install Dependencies

mvn clean install -DskipTests
3

Run Tests

# Run with browser visible
mvn test

# Run in headless mode
mvn test -Dheadless=true
4

View Test Reports

Test reports are generated in: target/surefire-reports/

GitHub Repository Setup

Initialize Git Repository

# Navigate to project directory
cd C:\ci-cd\demo-pipeline

# Initialize git
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: Login test automation with CI/CD"

Push to GitHub

# Create a new repository on GitHub first, then:
git remote add origin https://github.com/your-username/demo-pipeline.git
git branch -M main
git push -u origin main

GitHub Actions Pipeline

The CI/CD pipeline is automatically configured in .github/workflows/test.yml. It will:

  • Trigger on push to main or master branch
  • Trigger on pull requests
  • Set up Java 11 and Chrome browser
  • Run tests in headless mode
  • Upload test reports as artifacts
  • Display test results summary

Troubleshooting

Common Issues

Chrome driver version mismatch

WebDriverManager handles this automatically. If issues persist, update Chrome or clear the driver cache:

rm -rf ~/.cache/selenium

Tests timeout on slow connections

Increase wait times in BaseTest.java and LoginPage.java

GitHub Actions fails

Check the Actions tab in your repository for detailed logs. Common fixes:

  • Ensure headless mode is enabled for CI
  • Check Chrome installation in workflow
  • Verify network access to test URL
